## V0.10.0 — 多选与对齐/分布工具

### 功能描述

在 V0.9.0 基础上新增四项能力：

1. **画布框选**：启用 `selectionOnDrag`，在画布空白处拖拽矩形选择框选中范围内设备

3. **对齐工具（6 种）**：左对齐 / 水平居中 / 右对齐 / 顶对齐 / 垂直居中 / 底对齐，自动对齐到 10px 网格
4. **分布工具（2 种）**：水平等距分布 / 垂直等距分布，首尾节点锚定，中间节点均匀排布
5. **Escape 取消选中**：按 Esc 键清除所有选中节点和边

### 技术要点

- **框选**：`selectionOnDrag`（React Flow v12.11 内置）启用画布拖拽选择框；`selectionMode="partial"` 只需部分重叠即选中
- **`onSelectionChange` 与 `onNodeClick` 交互**：Shift+Click 时 `onNodeClick` 先于 `onSelectionChange` 触发。`onSelectionChange` 在 `count > 1` 时覆盖 `selectedNode = null`，React 在同一事件循环中批量更新 State，无闪烁
- **Escape 安全守卫**：跳过 `INPUT/TEXTAREA/SELECT` 焦点（与 InlineEdit 冲突），跳过打开中的确认对话框（`showNewConfirm/showOpenConfirm/showAutoSaveRecover`）
- **对齐到网格**：`snapToGridPoint(val) = Math.round(val / 10) * 10`，与 React Flow 的 `snapGrid={[10, 10]}` 一致
- **分布算法**：按轴排序 → 首尾锚定 → 中间等距：`spacing = (max - min) / (count - 1)` → `min + i * spacing`
- **撤销支持**：所有 8 个对齐/分布操作在修改前调用 `history.pushSnapshot(nodesRef.current, edges)`，Ctrl+Z 完全恢复
- **闭包安全**：使用 `nodesRef.current`（而非闭包中的 `nodes` state）读取最新选中节点，与 `isValidConnection` 中已建立的模式一致
- **按钮图标**：使用 Unicode 字符（⇤⇔⇥⇧⇳⇩↔↕），与 Toolbar 中现有 undo/redo 图标（↶↷）风格一致

## V0.10.0 Bug 修复（2026-07-01 后续）

### 修复的 Bug

1. **框选计数偏差**：`onSelectionChange` 之前忽略 React Flow v12 的 callback 参数 `{ nodes, edges }`，改用 `nodesRef.current`（在 Zustand store 更新后尚未来得及同步）。改为接收并使用 callback 参数中的 `selNodes.length`，直接来自 React Flow 内部 Zustand store，始终准确。

2. **右键批量删除菜单不出现**：React Flow v12 在多选节点上渲染 `NodesSelection` 覆盖层，拦截右键事件。`onNodeContextMenu` 不会触发。添加 `onSelectionContextMenu` prop 处理器，当用户在选中节点覆盖层上右键时显示批量删除菜单。

3. **对齐工具仅生效一次**：同轴操作（如左对齐后再右对齐）因所有节点已在同一位置而无法产生视觉变化，属于预期行为。跨轴操作正常。

### 技术要点

- **React Flow v12 选择覆盖层**：`NodesSelection` 组件在多选节点上方渲染半透明选择框，其 DOM 层级高于节点元素，拦截所有鼠标事件（包括右键）。需通过 `onSelectionContextMenu` prop 单独处理。
- **`onSelectionChange` 参数**：React Flow v12 传入 `{ nodes: Node[], edges: Edge[] }`，其中 `nodes` 来自内部 Zustand store（`nodeLookup`），始终与选择状态同步——比从 React `useNodesState` 读取更可靠。
- **`onNodesChangeWithSync` 中的 `selectedNode` 同步**：当 `selectedNode` 非 null 且 select changes 将其 `selected` 设为 false 时，保持非 null（设置 `{ ...prev, selected: false }`）而非清空。依赖 `onSelectionChange` 后续将其设为 null（多选时）或保持（单选时）。

## V0.11.0 — 用户体验打磨 + 新增功能

### 功能描述

在 V0.10.0 基础上新增以下功能：

**第1轮：复制粘贴 + 右键菜单增强**
1. **复制/粘贴节点 (Ctrl+C/V)**：支持画布设备复制粘贴，粘贴位置偏移 50px，可重复粘贴
2. **复制节点 (Ctrl+D)**：快速原地复制选中设备，偏移 30px
3. **画布右键菜单**：右键空白画布弹出菜单，支持粘贴/全选/适应视窗
4. **节点右键菜单增强**：新增「复制设备」选项
5. **批量右键菜单增强**：新增「复制选中设备」选项

**第2轮：拓扑模板 + 画布搜索 + 网格吸附切换**
6. **拓扑模板**：保存/加载/删除拓扑模板（存储于 userData/templates/）
7. **画布搜索 (Ctrl+F)**：浮动搜索栏，按设备名称/型号/厂商/描述/IP/备注检索，点击结果自动定位
8. **网格切换**：工具栏按钮控制点阵网格显示/隐藏
9. **吸附切换**：工具栏按钮控制 10px 吸附开关

**第3轮：节点分组 + 缩放控件 + 状态栏**
10. **节点分组 (Ctrl+G)**：为选中设备分配分组名称；Ctrl+Shift+G 取消分组
11. **缩放控件**：工具栏 +/- 按钮 + 缩放百分比 + 适应视窗按钮
12. **底部状态栏**：实时显示缩放比例、设备/连线计数、快捷键提示
