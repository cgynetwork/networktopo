# 2026-07-02 — 网络机柜绘制功能

## 概述
V1.1.0 核心功能：新增网络机柜绘制能力，将设备拓扑从平面扩展到机柜空间。

## 完成内容

### Phase 1: 数据模型
- 新增 `RackNodeData`、`RackDeviceNodeData`、`RackAccessory` 等 TypeScript 类型
- 新建 `src/renderer/utils/rackUtils.ts`：U 位常量、尺寸计算、空闲查找、吸附算法
- 数据库增量迁移：新增配线架、超融合、存储、运营商光猫 4 个类别 + 13 个设备型号

### Phase 2-3: 节点组件
- `RackNode.tsx`：机柜 SVG 渲染（金属框架 + U 标尺 + 导轨 + 附件）
- `RackDeviceNode.tsx`：双模式渲染（前面板/详细视图），复用 DeviceIllustration + ConnectionHandles

### Phase 4: 侧边栏
- 新增"网络机柜"拖拽区域，11 种标准尺寸可拖拽（MIME: `application/topo-rack`）

### Phase 5: 拖放交互
- 扩展 `onDrop`：三种分支（机柜/机柜内设备/独立设备）
- 新增 `onNodeDragStop`：机柜内设备 U 位吸附 + 冲突检测
- 删除机柜级联删除子设备

### Phase 7: 样式
- themes.css 新增 ~40 个机柜 CSS 变量（默认/鎏金双主题）
- 新增 4 个设备类别颜色变量

### Phase 8: 序列化
- TopoFile 版本升级至 1.1.0，支持 parentId/extent 序列化
- 加载兼容：旧文件正常回退

## 技术决策
- 机柜内设备使用 React Flow `parentId` + `extent: 'parent'` 机制
- 机柜附件（理线器/盲板/PDU）作为机柜内部数据，不创建 React Flow 节点
- 1U = 28px，紧凑模式宽 320px，详细模式宽 660px

## 文件清单
- 新建：rackUtils.ts, RackNode.tsx, RackDeviceNode.tsx
- 修改：types/index.ts, seed.ts, App.tsx, Sidebar.tsx, themes.css, useFileOperations.ts, CHANGELOG.md

  - `elbowHorizontalOffset` 控制中间垂直/水平段的位置偏移（左右）

- `AnimatedEdge.tsx` 新增悬停工具提示：
  - 鼠标进入 Edge 区域后启动 1.5s 定时器
  - 超时后通过 `EdgeLabelRenderer` 渲染说明内容（浮于 Edge 中点上方）
  - 鼠标离开时立即清除定时器并隐藏 Tooltip
  - 透明宽击区域（24px）提升悬停命中率

# V1.1.4 — 肘形路径重新设计 + 演示模式

## 肘形路径重新设计

### 问题
`elbowOffset`（肘形高度）与 `elbowHorizontalOffset`（肘形左右）视觉效果相同，因为旧的 3-段路径中两者均仅影响 `midX`。

### 修复
替换为 5-段正交路径设计：
- **elbowOffset** → 控制水平中段从 SY/TY 中心的垂直偏移（真正的"高度"）
- **elbowHorizontalOffset** → 控制从源端口出发的水平距离（"左右"调节）
- 两端始终固定于连接点 (sx,sy) 和 (tx,ty)
- 通过已有的 `stroke-linejoin: round` 实现自动圆角
- 垂直连接（Bottom↔Top）采用镜像逻辑，翻转坐标轴
- 混合方向回退至 React Flow 内置 `getSmoothStepPath`

# V1.1.6 — 肘形连接线反向动画修复 + 演示模式端口跟随修复

## 修复内容

### 1. 肘形连接线反向动画修复
- **根因**：反向动画路径生成时，`elbowHorizontalOffset` 被应用到原目标端口侧（交换后成为新的"源"），导致反向路径几何形状与正向不同，粒子动画轨迹异常
- **修复**：反向路径生成时传入 `elbowHorizontalOffset=0`，确保反向路径与正向路径几何完全一致，仅遍历方向相反
- 影响文件：`AnimatedEdge.tsx`

### 2. 演示模式端口固定点跟随修复
- **根因**：`useLayoutEffect` 测量 SVG 偏移的依赖数组为 `[nodeWidth, nodeHeight]`，切换演示模式时两者均不变（均为 auto-size），导致 `svgOffset` 不更新，端口连接把手仍按旧偏移量（含标题栏高度）定位
- **修复**：将 `isDemoMode` 加入 `useLayoutEffect` 依赖数组，确保演示模式切换时重新测量 SVG 位置
- 影响文件：`DeviceNode.tsx`

### 3. 演示模式设备 SVG 高度自适应
- **根因**：无端口设备（如无线接入点）在演示模式下 SVG 高度仍使用 `SVG_H_DEFAULT`（144px），下方留白过大
- **修复**：演示模式下无端口时使用 `SVG_H_MIN`（100px）替代 `SVG_H_DEFAULT`，使设备图形与框线紧贴
- 影响文件：`DeviceNode.tsx`

## 修改文件
- 修改：AnimatedEdge.tsx、DeviceNode.tsx

# V1.1.7 — 肘形连接线反向动画彻底修复 + 演示模式端口/线缆可见性修复

## 修复内容

### 1. 肘形连接线反向动画彻底修复
- **根因**：V1.1.6 的修复（反向路径 `elbowHorizontalOffset=0`）仍不正确 — 当用户设置了非零 `elbowHorizontalOffset` 时，正向路径首段延伸 45px（15 + 30），而反向路径首段仅延伸 15px（15 + 0），两条路径几何形状不同
- **修复**：不再重新生成反向路径，改用 SVG `keyPoints`/`keyTimes` 机制在**同一条 `edgePath`** 上反向驱动 `animateMotion`：
  - 正向：`keyPoints` 和 `keyTimes` 使用默认值（从 0 到 1）
  - 反向：`keyPoints="1;0"` `keyTimes="0;1"`，粒子从路径末端移动到起点
  - 辉光动画 (`stroke-dashoffset`) 反向时交换 `from`/`to` 值
  - 此方案适用于**所有路径样式**（肘形、贝塞尔、直线），零几何误差
- 影响文件：`AnimatedEdge.tsx`（移除 `animPath` 变量，新增 `isReverse` 标志，6 个 `animateMotion` + 1 个 `animate` 元素更新）

### 2. 演示模式端口固定接入点恢复可见
- **根因**：`showHandles` 在演示模式下强制为 `false`，且 `ConnectionHandles` 组件被 `!isDemoMode` 守卫完全移除
- **修复**：
  - `showHandles` 改为 `isDemoMode ? true : (selected || isHovered)`，演示模式下端口始终可见
  - 移除 `ConnectionHandles` 渲染的 `!isDemoMode` 守卫，仅保留 `svgOffset` 检查
- 影响文件：`DeviceNode.tsx`

### 3. 演示模式设计意图重述
演示模式下画布仅保留：
- ✅ 设备 SVG 图形
- ✅ 网络端口固定接入点（ConnectionHandles）
- ✅ 设备之间互联线缆（Edges）
- ❌ 标题栏、设备名称/型号/端口文字、分组徽章、业务提示框、NodeResizer（已裁剪）

# V1.1.8 — 演示模式设备外框紧贴 SVG

## 修复内容

### 演示模式设备外框自适应收缩
- **问题**：演示模式下，外层盒子宽度仍由端口数量决定（`getNodeWidth` 返回 200-720px），导致盒子远大于设备 SVG 图示，留白过多
- **修复**（经两轮迭代细化）：
  - `nodeWidth`：优先尊重手动拖拽尺寸（`rfWidth`），未手动调整时在演示模式下固定为 200px（仅用于 SVG 内部端口布局计算）
  - `nodeHeight`：演示模式下始终由 SVG 内容决定高度（无视手动调整的高度值）
  - 外层 `<div>` 的 `width`：演示模式下，仅当未手动调整时设为 `undefined`（自动收缩）；手动调整过则保留手动宽度
  - 外层 `<div>` 的 `height`：演示模式下始终 `undefined`（自动收缩至 SVG 内容高度）
- 最终行为：
  - 从未拖拽过：宽高均自动收缩至 SVG 大小
  - 手动拖拽过：宽度保留手动值，高度收缩至 SVG 内容高度
- 影响文件：`DeviceNode.tsx`

- 修改：DeviceNode.tsx
