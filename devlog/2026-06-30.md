## V0.8.1 — 线缆层级修复 + 端口标签可拖拽 + 肘形高度扩展

### 问题报告

用户验证 V0.8.0 后反馈三个问题：
1. **线缆被设备覆盖**：端口连接线缆渲染在设备节点之下，无法看清完整线缆路径
2. **端口标签被覆盖**：端口标签（如 GE1、SFP3）被设备节点遮挡
3. **肘形连接线高度范围不足**：当前最大 250px，需要扩展到 400px

### 修复

#### 1. React Flow 层级覆写
- [x] `global.css`：新增 `.react-flow__edges { z-index: 1000 !important }` 和 `.react-flow__edgelabelrenderer { z-index: 1001 !important }`
- 边路径 `fill: none` 仅描边，不拦截对下层节点的鼠标事件
- EdgeLabelRenderer 默认 `pointer-events: none`，仅显式设为 `auto` 的端口标签可交互

#### 2. 端口标签自由拖拽
- [x] `types/index.ts`：EdgeData 新增 4 个拖拽偏移字段 (`sourcePortOffsetX/Y`, `targetPortOffsetX/Y`)
- [x] `AnimatedEdge.tsx`：
  - 使用 `useReactFlow().setEdges` 在拖拽结束时提交偏移到 React Flow 状态
  - 本地 `useState` 管理拖拽中的视觉位置（避免 mousemove 触发全局重渲染）
  - `useRef` 跟踪拖拽基准点和当前偏移，解决闭包过期问题
  - 端口标签样式：移除 `pointer-events-none` → `cursor: grab`/`grabbing` + `select-none`
  - 拖拽偏移随 `.topo.json` 保存/加载，旧文件默认 `(0,0)` 向后兼容

#### 3. 肘形高度范围扩展
- [x] `PropertyPanel.tsx`：滑块 `max` 250 → 400，标签 `远(250px)` → `远(400px)`

### 技术要点
- 拖拽性能：mousemove 中仅更新本地 state（不触发 React Flow 全局渲染），mouseup 时一次性提交

- 层级安全：SVG path 元素 `fill: none` → 鼠标事件穿透，节点交互完全不受影响
- 向后兼容：旧 .topo.json 无偏移字段 → `undefined ?? 0 → 0`，标签显示在默认计算位置

## V0.8.2 — 端口标签不可见修复 + 肘形高度负值扩展

### 问题报告

V0.8.1 CSS z-index 覆写未生效，端口标签仍然不可见。肘形高度近方向需要下探到负值。

### 根因分析

`@xyflow/react` v12.11.1 中 EdgeLabelRenderer portal 的 CSS 类名为 `.react-flow__edgelabel-renderer`（**带连字符**），但 V0.8.1 覆写使用了 `.react-flow__edgelabelrenderer`（无连字符）—— 选择器不匹配，z-index 从未生效。

同时端口标签 `<div>` 缺少 `pointerEvents: 'all'`，在 EdgeLabelRenderer 默认 `pointer-events: none` 下无法接收鼠标事件。

| 选择器 | 状态 |
|--------|------|
| `.react-flow__edgelabelrenderer` | ❌ 不存在于 v12 |
| `.react-flow__edgelabel-renderer` | ✅ v12 实际类名 |
| `.react-flow__edges` | ✅ 一直正确 |

### 修复

#### 1. CSS 类名修正
- [x] `global.css`：`.react-flow__edgelabelrenderer` → `.react-flow__edgelabel-renderer`

#### 2. 端口标签 pointerEvents
- [x] `AnimatedEdge.tsx`：源/目标端口标签 style 新增 `pointerEvents: 'all'`（EdgeLabelRenderer portal 默认 `pointer-events: none`，子元素需显式覆盖）

#### 3. 肘形高度负值扩展
- [x] `PropertyPanel.tsx`：滑块 `min` 10 → -200，标签 `近(10px)` → `近(-200px)`
- [x] `types/index.ts`：`elbowOffset` 注释 10-400px → (-200)-400px

### 变更文件

| `src/renderer/components/PropertyPanel/PropertyPanel.tsx` | 修改（滑块 min -200） |

## V0.8.3 — 端口连接点精准定位 + 带宽标签 SVG 渲染 + 肘形高度恢复

### 问题报告

1. **缩放节点时连接点偏移**：调整设备横纵大小后，端口 Handle 位置漂移
2. **带宽标签被线缆覆盖**：带宽值在画布中显示在线缆之下
3. **连线端点未紧贴端口**：边端点与端口中心有可见偏移（~3.5px）
4. **肘形高度近方向**：-200px 不符合预期，恢复至 10px

### 根因分析

| 问题 | 根因 |
|------|------|
| 连接点缩放偏移 | `useLayoutEffect` 用 `svgContainerRef`（容器 div）测量 svgOffset，但容器有 `px-2` padding + `flex justify-center` 居中，SVG 元素起点与容器边缘有偏移 |
| 带宽标签被覆盖 | 带宽标签走 `EdgeLabelRenderer` HTML portal，尽管 z-index 1001 > edges 1000，但 SVG `<path>` stroke 仍视觉覆盖 HTML 文本 |
| 端点偏移 | `Handle` 使用 `position={Position.Top}`，React Flow 将连接点置于 Handle **顶部边缘**而非中心（偏移 = handleSize/2 ≈ 3.5px） |

### 修复

#### 1. SVG 元素直接测量
- [x] `DeviceNode.tsx`：`svgContainerRef.current.querySelector('svg')` 直接获取 SVG 元素的 getBoundingClientRect，自动消除 padding/centering 偏移

#### 2. 带宽标签改为 SVG `<text>`
- [x] `AnimatedEdge.tsx`：移除 `EdgeLabelRenderer` 包裹，改为 SVG `<text>` 元素渲染，置于 `<BaseEdge>` 之后 → 同一 SVG 层内后者覆盖前者

#### 3. Handle Position.Top 补偿
- [x] `ConnectionHandles.tsx`：`adjustedTop = handleY + handleSize / 2`，向下偏移半个 Handle 尺寸，使 `Position.Top` 连接点对准端口中心

#### 4. 肘形高度恢复
- [x] `PropertyPanel.tsx`：`min="-200"` → `min="10"`，标签恢复 `近(10px)`
- [x] `types/index.ts`：`elbowOffset` 注释恢复 10-400px

### 变更文件

| 文件 | 操作 |
|------|------|
| `src/renderer/components/nodes/DeviceNode.tsx` | 修改（svgOffset 测量 SVG 元素本身） |
| `src/renderer/components/edges/AnimatedEdge.tsx` | 修改（带宽标签 EdgeLabelRenderer → SVG text） |
| `src/renderer/components/nodes/ConnectionHandles.tsx` | 修改（Handle Position.Top 偏移补偿） |

## V0.8.4 — 连线端点精准修复 + 带宽标签 SVG 光环可读性

### 问题报告

V0.8.3 修复后验证仍存在两个问题：
1. **连线端点未紧贴端口中心**：边端点与端口中心仍有可见偏移
2. **带宽标签被线缆覆盖**：带宽值在画布中显示不够清晰，视觉上仍被电缆遮挡

### 根因分析

| 问题 | 根因 |
|------|------|
| 端点偏移 | V0.8.3 的 `adjustedTop` 补偿数学正确，但 React Flow 的 `getHandleBounds` 在节点挂载/尺寸变化时测量 Handle DOM 位置（`getBoundingClientRect`）。我们的 `useLayoutEffect` 更新 `svgOffset` → Handle 重新渲染到正确位置，但 React Flow **不会**自动重新测量 — 内部 `handleBounds` 保持过期数据。`getHandlePosition` 使用过期数据计算边端点坐标，产生偏移。 |
| 带宽标签被覆盖 | V0.8.3 将带宽标签从 `EdgeLabelRenderer` HTML portal 改为 SVG `<text>` 元素，在 SVG 绘制顺序中确实位于 `<BaseEdge>` 之上，但 9px 灰色文字缺少背景/轮廓，与下方线缆重叠时人眼难以辨识。 |

### 修复

#### 1. React Flow Handle 强制重测
- [x] `DeviceNode.tsx`：引入 `useUpdateNodeInternals`，在 `svgOffset` 更新后的 `useLayoutEffect` 中调用 `updateNodeInternals(id)`
- `useUpdateNodeInternals` 内部使用 `requestAnimationFrame` 延时，确保 Handle DOM 位置更新完成后在下一帧测量
- 此时 `adjustedTop`（Position.Top 补偿）正确生效：连接点精准对准端口中心

#### 2. 带宽标签 SVG 文本光环
- [x] `AnimatedEdge.tsx`：为带宽 `<text>` 添加 `paintOrder="stroke fill"` + `stroke="var(--color-canvas)"` + `strokeWidth={3.5}`
- `paintOrder="stroke fill"` 先绘制描边再绘制填充 → 描边在文字下方形成底色光环
- 光环颜色 `--color-canvas`（#FFFFFF 默认 / #0B1121 鎏金）确保在任何颜色线缆上方清晰可辨

#### 3. 版本号同步
- [x] `Toolbar.tsx`：V0.6.0 → V0.8.4
- [x] `package.json`：0.7.0 → 0.8.4

### 变更文件

| 文件 | 操作 |
|------|------|
| `src/renderer/components/nodes/DeviceNode.tsx` | 修改（新增 useUpdateNodeInternals 调用） |
| `src/renderer/components/edges/AnimatedEdge.tsx` | 修改（带宽标签添加 paintOrder 光环） |
| `src/renderer/components/Toolbar/Toolbar.tsx` | 修改（版本号 V0.6.0 → V0.8.4） |
| `package.json` | 修改（版本号 0.7.0 → 0.8.4） |

**4 个修改文件。**

### 技术要点
- React Flow 的 `getHandleBounds` 测量时机：仅在节点挂载、尺寸变化、手动 `updateNodeInternals` 时触发
- `useUpdateNodeInternals` 内部使用 `requestAnimationFrame` → 安全地在 `useLayoutEffect` 中调用
- SVG `paintOrder` 控制填充/描边绘制顺序：`stroke fill` = 描边在下，填充在上 → 文字光环效果
- `--color-canvas` 是画布背景色 → 光环始终与背景同色，在任意主题下通用
