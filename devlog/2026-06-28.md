### V0.1.1 — GIF 清晰度优化 & 动画保留 & 连接形式切换

**新增功能**:
- [x] 连线连接形式切换（右键菜单 + 属性面板）：自适应连接（默认）、直线连接、肘形连接线
- [x] `PathStyle` 类型 (`'adaptive' | 'straight' | 'step'`) 添加到 EdgeData

- [x] 路径反转函数支持直线和曲线两种格式

**缺陷修复**:
- [x] **关键修复**: GIF 清晰度极低 → 旧方案使用 `pixelRatio: 1` + `scale: 0.6`，画面模糊
- [x] **关键修复**: GIF 动画效果丢失 → `html-to-image` 的 `toPng` 通过 DOM 克隆 + SVG foreignObject 渲染，无法保留 `<animateMotion>` 的实时变换状态
- [x] 切换到 Electron `webContents.capturePage()` API 捕获帧，该 API 截取实际渲染输出（含 SVG 动画当前位置），从根本上解决动画丢失问题
- [x] 新增 `capture:frame` IPC 处理器（main.js + file-handlers.ts 同步）
- [x] GIF 编码质量提升: quality 3（原 8）、dither 开启、40 帧（原 20）、max 900px

**技术要点**:
- `capturePage()` 返回 `NativeImage`，通过 `toDataURL()` 转为 PNG data URL 传回渲染进程
- `capturePage` 坐标使用 CSS 像素（DIP），不需要手动乘以 DPR
- `getStraightPath` 返回 `M sx sy L ex ey` 格式，`getSmoothStepPath` 返回与 bezier 相同的 C 曲线格式
- `getSmoothStepPath` 的 borderRadius 默认 25px，可通过 options 参数调整

**V0.1.1 缺陷修复 (2026-06-28 续)**:
- [x] **关键修复**: 连线连接形式功能导致线缆无法显示 → `AnimatedEdge.tsx` 中 `getBezierPath`/`getStraightPath`/`getSmoothStepPath` 使用对象参数而非位置参数
- [x] **关键修复**: 方向设为反向时动画效果丢失/异常 → 删除 `reversePath()` 字符串反转函数，改用路径函数交换 source/target 坐标生成反向动画路径
- [x] **端口号改进**: 将 `portLabel` 拆分为 `sourcePort`（本端端口号）+ `targetPort`（对端端口号），分别显示在线缆两端（源端蓝标、目标端橙标）
- [x] **自定义设备功能**: 侧边栏新增「＋ 自定义设备」按钮，弹窗表单支持选择分类、选择/新建厂商、填写型号/描述/端口信息，创建后自动归入设备分类并支持拖拽到画布
- [x] 新增 `db:getVendors` + `db:addVendor` IPC 处理器（main.js + device-handlers.ts）
- [x] 新增 `AddDeviceModal` 组件，含分类下拉、厂商下拉/新建、型号/描述/端口输入

**技术要点**:
- `@xyflow/react` v12 中 `getBezierPath`/`getStraightPath`/`getSmoothStepPath` 签名均为 `({ sourceX, sourceY, ... })` 对象参数形式
- `getSmoothStepPath` 返回的路径含多段直线和弧线（`L ... A ...`），不是简单贝塞尔曲线（`C ...`），无法通过正则交换控制点实现反转
- 端口标签通过 `EdgeLabelRenderer` 分别定位到 source/target 端，偏移量 `Math.min(28, dist * 0.15)`
- 正确做法：反向时直接调用路径函数，传入交换后的 source/target 坐标，生成从 target 到 source 的有效 SVG 路径
- `generatePath()` 辅助函数统一三种路径样式的调用，`edgePath`（视觉线）始终 source→target，`animPath`（动画路径）按 direction 决定方向

### V0.1.0 技术要点（存档）
- React Flow Handle 必须是节点根元素的 DIRECT 子元素，不能包裹在 div 中
- 每个设备节点 8 个 Handle（4 方向 × source+target），offset 避免重叠
- gif.js 通过 Vite `?url` 导入 worker 脚本，需 `base: './'` 确保 Electron file:// 协议路径正确
- 项目存在两套主进程：`main.js`（npm start 使用）和 `src/main/`（TypeScript 编译输出），需同步更新

## 关键问题记录
1. **ELECTRON_RUN_AS_NODE**: VS Code 设置该变量导致 Electron 以纯 Node 模式运行 → 所有 npm scripts 已自动清除
2. **PostCSS BOM**: package.json 包含 UTF-8 BOM → 已重写所有配置文件
3. **better-sqlite3 版本不匹配**: 系统 Node 24 vs Electron Node 20 → @electron/rebuild 解决
4. **electron-vite 打包原生模块**: main 进程打包 better-sqlite3 失败 → 使用 main.js 作为入口
5. **React Flow Handle 必须是直接子元素**: 包裹在 div 中导致 React Flow 无法检测 → 8 个 Handle 直接放在节点根元素
6. **React Flow 连接需要 target Handle**: 只有 source Handle 时 React Flow 静默拒绝连接 → 添加 source+target 对
7. **gif.js worker 在 Electron 中加载失败**: `base: '/'` 导致 `file://` 协议下绝对路径解析错误 → `base: './'` 使用相对路径
8. **main.js 与 TS 源不同步**: `npm start` 使用 main.js，需手动同步 IPC 处理器
