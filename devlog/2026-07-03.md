# 2026-07-03 — V1.5.0 业务图片移动钳制修复

## 修复

### 图片移动限制修复

**问题**：业务图片在"互联网应用"机箱内只能向左和向上移动（偏移被钳制到 ≤0），上传多张图片时无法自由排列，导致图片重叠覆盖。

**根因**：`DeviceNode.tsx` 的 `onMouseMove` 中使用 `Math.min(0, newX)` 和 `Math.min(0, newY)` 将偏移硬钳制为非正数。

**修复**：将钳制改为比例边界：
```typescript
// 左边缘：图片可向左移动到仅剩 10px 可见
const minX = -(originalW * img.scale - 10)
// 右边缘：图片可向右移动到 imgAreaW - 10 处
const maxX = imgAreaW - 10
const clampedX = Math.max(minX, Math.min(maxX, newX))
```

### 修改文件

| 文件 | 变更 |
|------|------|
| `src/renderer/components/nodes/DeviceNode.tsx` | onMouseMove/onMouseUp 中 offset 钳制逻辑改为比例边界 |

# 2026-07-03 — V1.5.1 四项 UX 增强

## 新增功能

### 1. 业务图片缩放把手 hover 显隐

**问题**：缩放把手常驻显示影响美观，遮挡机箱内部细节。

**方案**：纯 CSS 方案，零 JS 改动：
```css
.internetapp-resize-handle { opacity: 0; transition: opacity 0.15s ease; }
.internetapp-image:hover + .internetapp-resize-handle,
.internetapp-resize-handle:hover { opacity: 1; }
```

### 2. 连线接口 IP 配置 + 可拖拽标签

**问题**：连线缺少接口 IP 配置能力，无法在拓扑图中标注 IP 地址。

**方案**：参照现有端口标签的拖拽模式：
- **EdgeData 扩展**：`sourceIp`/`targetIp` + 4 个 offset 字段
- **IP 标签渲染**：EdgeLabelRenderer 中独立 `<div>`，位于端口标签下方 14px，斜体 + 配色区分
- **拖拽持久化**：mousedown/mousemove/mouseup + setEdges 模式（与端口标签一致）
- **属性面板**：本端/对端两个文本输入框，"🌐 接口 IP" 分区
- **CSS 变量**：本端绿色系（`--color-ip-source-*`）、对端紫色系（`--color-ip-target-*`），支持双主题

### 3. AP 设备端口居中

**问题**：无线接入点设备端口整体偏左，与 AP 机箱的居中布局不匹配。

**根因**：`getDeviceLayoutParams('无线接入点')` 返回 `panelX: 0`，且 `ApSvg` 未解构传递 `panelX`。

**修复**：
- `panelX` 从 `0` 改为 `bodyX = Math.round(svgW * 0.18)`（匹配机箱左边缘）
- `ApSvg` 解构 `panelX` 并传递到 `<PortPanel panelX={panelX}>`

### 4. SDWAN CPE Tunnel 端口数量可编辑

**问题**：Tunnel 端口硬编码为 2 个，无法按需配置。

**方案**：
- **NodeData 扩展**：`tunnelPortCount?: number`（默认 2，范围 1-8）
- **属性面板**：隧道端口开关下方 +/− 按钮调节数量
- **动态 Handle 生成**：`for (i=0; i<count; i++) TUNNEL-{i+1}`
- **SVG 自适应**：`tunnelW = Math.min(22, panelBodyW / count - 6)`，超出时缩小宽度
- **多行布局**：count > 4 时自动分行（maxPerRow=4）
- **连接验证**：`startsWith('TUNNEL-')` 通配任意编号（无需修改）
