## V0.7 — 问题修复：端口布局算法重写 + SVG 响应式高度

### 问题

1. **端口布局异常**：端口按数量均分行（`portsPerRow`），不按实际端口宽度填行，导致行尾留有大量空白或端口溢出可视区域
2. **行换行条件有 bug**：`portIndexInRow >= 4` 守卫阻止端口不足 4 个时换行，导致宽端口（QSFP 28px）大量堆积溢出
3. **端口组间隔在换行后不重置**：组间 8px 间隔在行末累加，加速溢出
4. **SVG 高度固定 144px**：纵向缩放设备节点时端口面板不随之扩展/收缩
5. **端口面板不居中**：`yOffset` 固定值，端口随设备高度变化时无法保持在机壳中央区域

### 修复

#### 1. `portParser.ts` — `getPortLayout()` 算法重写

**旧算法**：
- 按 `Math.ceil(totalPorts / rows)` 均分每行端口数
- 行溢出条件需要 `portIndexInRow >= 4`（阻止少端口时换行）
- 端口不居中

**新算法**：
- **按宽度填行**：逐个端口检查 `cursorX + gap + portW > usableW`，放不下则自然换行
- 端口间距 PORT_GAP=3px，组间距 GROUP_GAP=10px
- **每行端口水平居中**：计算 `rowOffsetX` 将整行端口居中于可用区域
- **行数完全由内容决定**：不再预设行数，端口放完自然得出实际行数
- **端口面板垂直居中**：计算 `totalPanelH` 后，`topOffset` 将端口区居中于 `availableHeight`

#### 2. `DeviceIllustration.tsx` — SVG 高度响应式

- 所有 SVG 变体（Switch/Firewall/AC/AP/Server）新增 `svgH` prop
- 机身高度 `bodyH = svgH - 16` 动态适配
- `SVG_H` 常量 → `SVG_H_DEFAULT`（默认值）+ `SVG_H_MIN`（最小值 100）
- `PortPanel` 参数重构：
  - `yOffset` → `bodyH` + `bodyY`（基于机身坐标定位）
  - 新增 `panelX`（左侧面板偏移，如防火墙/AC 的安全/信号区）
  - 端口组使用 SVG `<g transform>` 统一偏移
- `ScrewHoles` 螺丝孔根据 `svgH` 定位底部孔位
- 服务器硬盘托架、驱动器 LED 使用比例计算公式适配动态高度

#### 3. `DeviceNode.tsx` — 传递动态高度

- 导入 `SVG_H_DEFAULT` 常量
- 计算 `svgH`：`nodeHeight ? Math.max(100, nodeHeight - 102) : SVG_H_DEFAULT`
  - 预留：header(~30px) + info(~60px) + padding(~12px) = ~102px
- 纵向缩放节点时 SVG 插图自动伸缩，端口始终保持在合理位置

### 变更文件

| 文件 | 操作 |
|------|------|
| `src/renderer/utils/portParser.ts` | 重写 `getPortLayout()` |
| `src/renderer/components/nodes/DeviceIllustration.tsx` | 重构（响应式高度、PortPanel API 变更） |
| `src/renderer/components/nodes/DeviceNode.tsx` | 修改（计算并传递 svgH） |

**3 个修改文件。**

### 技术要点
- 布局算法核心思路：先收集所有端口 → 逐行填充（宽度驱动）→ 每行居中 → 整体垂直居中
- `getPortLayout` 不再需要 `availableHeight` 来预设行数，而是用 `rowHeight` 和 `ROW_GAP` 约束单行高度后自然排布
- SVG `transform="translate(x, y)"` 替代逐端口 `yOffset` 加法，减少计算并提升可读性
- `RenderedPort.height` 由布局算法计算（而非硬编码 `getPortHeight`），保证端口高度与行高一致

## V0.7.0 — 问题修复：光纤端口左右分列布局 (Side-by-Side)

### 问题

当网络端口（铜缆 RJ45）与光纤端口（SFP/SFP+）并存时，光纤端口被垂直置于两行网络端口中间位置，而非水平排列在网络端口右侧。用户期望的布局为：网络端口分布至设备左侧，光纤端口分布于网络端口右侧，两者左右并列。

### 修复

**`portParser.ts` — `getPortLayout()` 布局算法重写**：

1. **左右分列布局**：当铜缆和光纤端口同时存在时，水平划分可用区域 — 铜缆左侧、光纤右侧，各自独立按分行规则填行
2. **宽度按端口数量比例分配**：`leftShareW = (usableW - SIDE_GAP) × leftTotal / totalBoth`
3. **对调逻辑保留**：当光纤端口数量 > 网络端口数量时，光纤移至左侧、网络端口移至右侧（光纤为主设备）
4. **仅一类端口存在时**：使用全宽布局（与旧版兼容）
5. **行对齐**：行列较少的侧在垂直方向上居中于行列较多的侧
6. **Mgmt 管理口**：始终置于最底部，使用全宽

**`DeviceNode.tsx` — 节点宽度调整**：适配左右分列布局所需额外水平空间（SIDE_GAP=10px）：
- ≤8 端口: 200px（不变）
- ≤16 端口: 280 → 320px
- ≤24 端口: 340 → 420px
- ≤48 端口: 600 → 640px
- >48 端口: 660 → 700px

**保留的已有功能**：
- `determineMaxPerRow(total)` 分行规则：≤12→max12, >12≤16→max8, >16≤24→max12, >24≤48→max24
- `fillRows()` + `positionRow()` 两级布局：count-driven 分行 + 行内 gap/port 压缩
- 万兆光纤标签 SFP+/SFP28（V0.7.1 修正）
- 模块化端口编辑（V0.7.1）

### 变更文件

| 文件 | 操作 |
|------|------|
| `src/renderer/utils/portParser.ts` | 重写 `getPortLayout`（左右分列布局替换上下交错布局） |
| `src/renderer/components/nodes/DeviceNode.tsx` | 更新 `getNodeWidth`（适配左右分列） |

**2 个修改文件。**

### 技术要点
- 左右分列布局核心：`leftRows = fillRows(leftItems, leftShareW, ...)` + `rightRows = fillRows(rightItems, rightShareW, ...)` → 合并同索引行 → 右侧 item.x 偏移 `rightBaseX`
- `positionRow` 已在各自分配的宽度内完成水平居中，合并时右侧仅需添加水平偏移
- 行列数较少的侧通过 `(maxRows - sideRows.length) / 2` 计算上下 padding 实现垂直居中
- 无铜缆或无光纤时回退到单类型全宽布局，与旧版行为一致
