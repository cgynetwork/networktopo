# 开发日志 — 2026-06-29

## V0.4.0 — 撤销/重做 + 最近文件 + 自动保存

### 新增功能

**撤销/重做 (Undo/Redo)**:
- [x] 快照式历史栈 `useHistory` hook（最大 50 步，JSON 深拷贝）
- [x] 在所有状态变更操作前自动记录快照：onConnect、onDrop、updateNodeData、updateEdgeData、handleDeleteEdge、handleDeleteNode、handleEdgePathStyle、handleOpen、menu deleteSelected、onNodesChange (remove)、onEdgesChange (remove)
- [x] 拖拽防抖：`isDraggingRef` 仅在拖拽开始时快照，中间帧跳过
- [x] 滑块防抖：`sliderSnapshotRef` Map 跟踪正在编辑的边，仅首次记录，400ms 空闲后重置
- [x] 尺寸调整防抖：仅在 `resizing === true` 时快照
- [x] 撤销支持到空画布（初始状态），重做恢复到最新状态
- [x] 新建/打开文件时自动清空历史栈
- [x] 已有 Ctrl+Z/Ctrl+Y 菜单快捷键（Edit 菜单）

**最近文件 (Recent Files)**:
- [x] 主进程维护 `userData/recent-files.json`，最多 10 条，含 filePath/name/timestamp
- [x] 动态 File 菜单：保存/打开文件时自动添加记录，`rebuildRecentMenu()` 重建子菜单
- [x] File 菜单「最近打开的文件」区域：分隔线 + 文件列表 + 结束标记，置于「退出」前
- [x] 点击最近文件菜单项直接打开文件（`openFileByPath` IPC）
- [x] 重复文件自动去重并移到列表顶部
- [x] IPC 通道：`file:getRecent`、`file:addRecent`、`file:clearRecent`、`file:openByPath`

**菜单 action 格式升级**:
- [x] `sendToRenderer` 支持 `string`（旧格式）和 `{ action, filePath? }`（新格式）两种负载
- [x] 向后兼容：所有旧菜单项无需修改
- [x] openRecent 通过对象格式携带 filePath

**自动保存 (Auto-Save)**:
- [x] `isDirty` 跟踪：所有状态变更操作设为 true，保存/打开/清空画布设为 false
- [x] 2 分钟定时器：有 `currentFilePath` 时静默保存，无路径时写入 `autosave.topo.json`
- [x] 启动时检测自动保存文件，弹出确认对话框提示恢复
- [x] 手动保存后清除自动保存文件
- [x] IPC 通道：`autoSave:write`、`autoSave:check`、`autoSave:clear`

**UI 改进**:
- [x] Toolbar 标题栏显示 `•` 未保存标记（isDirty）
- [x] 窗口标题更新为 `Topo V0.4.0`

- 快照使用 `JSON.parse(JSON.stringify(nodes/edges))` 深拷贝避免引用问题
- `main.js`（CommonJS）和 `src/main/index.ts`（TypeScript）必须同步更新所有 IPC handler 和菜单变更
- 动态菜单通过 label 标记（'recent-separator'、'recent-files-end'）定位和替换最近文件区域
- 滑块防抖关键：`sliderSnapshotRef` 是一个 Map，edgeId 作为 key，存在即表示正在连续变更中

## V0.4.5 — 双主题系统 (简洁 + 鎏金)

### 新增功能
- [x] 双主题支持：`default`（当前简洁白底蓝调）和 `gilded`（鎏金暗黑金调）
- [x] CSS 变量驱动：所有颜色通过 CSS 自定义属性定义，`data-theme` 属性一键切换
- [x] ThemeContext (`src/renderer/context/ThemeContext.tsx`)：React Context + localStorage 持久化
- [x] Toolbar 主题切换按钮：✨ 图标切换至鎏金，☀️ 图标切换回简洁
- [x] 主题切换即时生效，无需刷新

| 文字次色 | `#6B7280` | `#B8A890` 哑金 |

### 架构
```
src/renderer/
├── context/
│   └── ThemeContext.tsx       # 新增：主题状态管理 + localStorage
├── styles/
│   ├── global.css             # 修改：引入 themes.css，使用 CSS 变量
│   └── themes.css             # 新增：:root 默认主题 + [data-theme="gilded"] 鎏金主题
```
- Tailwind `colors` 全部映射到 CSS 变量 (`var(--color-*)`)
- 新增 Tailwind 颜色：`surface`、`danger-bg`、`amber-bg`
- 组件中 `bg-white` → `bg-surface`：Toolbar、Sidebar、PropertyPanel、DeviceNode、AnimatedEdge、AddDeviceModal
- `bg-canvas` 用于画布和 App 外壳（自动跟随主题）
- React Flow MiniMap 背景/边框跟随主题

### 技术要点
- `data-theme` 属性设置在 `<html>` 根元素，所有 CSS 变量通过属性选择器 `[data-theme="gilded"]` 覆写
- 主题偏好通过 `localStorage` 持久化，启动时自动恢复
- 鎏金主题的 `accent-color` 和滚动条颜色通过纯 CSS 覆写，无需修改 JS 代码
- 设备 SVG 插图保留原色（`#F8FAFC` 等），在浅色卡片内不受影响
- 所有 `bg-white` → `bg-surface` 替换保持语义化，`bg-canvas` 用于大面积背景
