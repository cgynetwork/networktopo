# 2026-07-08 — V1.5.0 导出修复 + 版本发布

## 导出功能完全重写

### 问题回顾
经过三轮迭代，导出功能积累了一个致命 bug：
- R1: `zoomFactor=2.0` + `capturePage(rect)` — 颜色正确但坐标系错位，导出含 UI 界面且拓扑不完整
- R2: 移除 zoomFactor + CSS 隐藏叠加层 — UI 问题解决但清晰度差、未裁剪空白
- R3: 恢复 zoomFactor + `executeJavaScript` 重算坐标 + `cropImage` 裁剪 — 坐标错位导致裁剪区域偏移数百像素，导出内容完全错误

### 根因分析
`cropImage()` 使用 1x 坐标系的 `reactFlowRect` 和 `contentScreenBounds` 来裁剪 2x 坐标系下截取的图片。`.react-flow` 元素在两种 zoom 级别下的 viewport 位置不同（sidebar/toolbar CSS 尺寸随 zoomFactor 缩放），导致裁剪偏移约 `flowToScreenPosition().x * DPR` 像素。

### 修复方案
完全移除 `zoomFactor` 操作，统一坐标系统：
1. 渲染进程在统一 zoom 级别计算 rect + contentScreenBounds
2. 主进程直接 `capturePage(rect)`，不改变 zoomFactor
3. `cropImage` 在同一坐标系下裁剪（坐标始终一致）
